B) DATA TYPES — RECOMMENDED DECLARATIONS
============================================================
Use these as your per-category selections in Play Console.

1) Personal info → Email address
- Collected: YES
- Shared: YES (service providers used to operate account/auth stack)
- Required or optional: Required for account features
- Purposes: App functionality, account management, security/fraud prevention

2) Financial info → Purchase history / transaction info (metadata)
- Collected: YES (payment status, transaction IDs, credit/stake metadata)
- Shared: YES (payment processor)
- Required or optional: Optional (only for paid/stake features)
- Purposes: App functionality, payments, fraud prevention/compliance
- Note: Full card data is not stored by Lockstep

3) Audio files → Voice notes
- Collected: YES (only when user initiates recording/upload)
- Shared: YES (transcription/AI processing provider and storage provider)
- Required or optional: Optional (voice features only)
- Purposes: App functionality

4) App activity → In-app interactions
- Collected: YES
- Shared: YES (infrastructure/processing providers)
- Required or optional: Required for service operation/reliability
- Purposes: App functionality, analytics, fraud prevention/security

5) App info and performance → Diagnostics/logs
- Collected: YES
- Shared: YES (infrastructure/processing providers)
- Required or optional: Required for reliability/security
- Purposes: Analytics, fraud prevention/security, app stability

6) User-generated content → Commitments, reflections, journal entries, preferences
- Collected: YES
- Shared: YES (storage/processing providers)
- Required or optional: Required for core app behavior once account is used
- Purposes: App functionality

7) Other user content / connected source metadata (e.g., calendar integration)
- Collected: YES (only if user enables integration)
- Shared: YES (authorized integration partners + infrastructure providers)
- Required or optional: Optional
- Purposes: App functionality
