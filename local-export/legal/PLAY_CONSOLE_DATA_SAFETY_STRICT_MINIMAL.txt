============================================================
B) DATA TYPES — MINIMAL DECLARATIONS
============================================================
1) Personal info → Email address
- Collected: YES
- Shared: NO (use NO only if no non-exempt third-party sharing under Play definitions)
- Required or optional: Required for account features
- Purposes: App functionality, account management, security/fraud prevention

2) User-generated content → Commitments, reflections, journal entries, preferences
- Collected: YES
- Shared: NO (use NO only if no non-exempt third-party sharing)
- Required or optional: Required after account use
- Purposes: App functionality

3) Audio files → Voice notes
- Collected: YES (only when user initiates recording/upload)
- Shared: NO for default build; YES if transcription/provider transfer occurs
- Required or optional: Optional
- Purposes: App functionality

4) Financial info → Purchase history / transaction metadata
- Collected: YES only if paid/stake features enabled in production
- Shared: YES if payment processor is used for live transactions
- Required or optional: Optional
- Purposes: App functionality, payments, fraud prevention/compliance

5) App activity → In-app interactions
- Collected: YES (operational telemetry only)
- Shared: NO (unless sent to non-exempt third parties)
- Required or optional: Mixed
- Purposes: App functionality, security

6) App info and performance → Diagnostics/logs
- Collected: YES
- Shared: NO (unless external crash/diagnostic processor is active)
- Required or optional: Mixed
- Purposes: App stability, security

7) Connected source metadata (e.g., calendar)
- Collected: YES only if user enables integration
- Shared: YES to authorized integration provider(s)
- Required or optional: Optional
- Purposes: App functionality
