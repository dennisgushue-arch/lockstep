## 1) App Store Copy (App Store + Google Play)

### Brand Positioning

- **Brand promise:** Lockstep helps people keep commitments by adding real stakes and clear consequences.
- **One-liner:** **Commit or pay.**
- **Value proposition:** Lockstep turns intentions into action by attaching real financial stakes to your goals.

### iOS App Store Fields

- **Subtitle:** Real-money accountability for goals

### Google Play Fields

- **App Name:** Lockstep: Commit or Pay
- **Short Description:** Turn your goals into non-negotiables with real financial stakes.

### Long Description (App Store + Play)

Stop breaking promises to yourself.

Lockstep is an accountability app that helps you follow through by putting real money behind your commitments.

How it works:

1. Set an intention — define what you’ll do and by when.
2. Lock in a stake — choose an amount you’re willing to lose.
3. Take action — stay accountable with deadlines and reminders.
4. Outcome happens — complete your commitment and keep your money; miss it and your stake is charged.

Why Lockstep works:

- Real stakes create real motivation
- Clear commitment flow without complexity
- Simple dashboard for active and completed pacts
- Secure payment handling via Stripe

Best for:

- Fitness and health goals
- Study and exam prep
- Creator consistency
- Personal projects
- Habit resets

**Call to action:** Download Lockstep and make your next goal non-negotiable.

## 4) V2 Channel-Tailored Ads (Performance vs Brand)

### How to Use This Section

- **Performance ads:** Optimize for installs and first commitment creation.
- **Brand ads:** Optimize for awareness, recall, and message resonance.
- Keep attribution windows and creative tests separate by objective.

### Creative ID Naming Convention

Use this format for production asset tracking:

`LKS_[CHANNEL]_[OBJECTIVE]_[NUMBER]`

Examples:

- `LKS_TT_PERF_01` = Lockstep, TikTok, Performance, creative 01
- `LKS_IG_BRAND_01` = Lockstep, Instagram Reels, Brand, creative 01
- `LKS_YT_PERF_01` = Lockstep, YouTube Shorts, Performance, creative 01

Recommended channel codes:

- `TT` = TikTok
- `IG` = Instagram Reels
- `YT` = YouTube Shorts

Recommended objective codes:

- `PERF` = Performance
- `BRAND` = Brand

### Asset Filename Convention

Use this format for production assets:

`[CREATIVE_ID]_v[VERSION]_[VARIANT].[EXTENSION]`

Examples:

- `LKS_TT_PERF_01_v1_hookA.mp4`
- `LKS_TT_PERF_01_v2_hookB.mp4`
- `LKS_IG_BRAND_01_v1_master.mp4`
- `LKS_YT_PERF_01_v1_thumbA.png`
- `LKS_YT_PERF_01_v1_captions.srt`

Recommended variant labels:

- `master` = final approved master export
- `hookA`, `hookB`, `hookC` = opening-hook variants
- `cut15`, `cut30` = duration-specific exports
- `thumbA`, `thumbB` = thumbnail variants
- `captions` = subtitle file
- `square`, `vertical`, `story` = aspect-ratio export

Recommended extensions by asset type:

- `.mp4` = final video export
- `.mov` = editable review/export file
- `.png` = thumbnail or poster frame
- `.srt` = captions
- `.csv` = metadata handoff
- `.fig` or source-tool native formats = design source file

Suggested production examples:

- `LKS_TT_PERF_01_v1_master.mp4`
- `LKS_TT_PERF_01_v1_hookA.mp4`
- `LKS_TT_PERF_01_v1_thumbA.png`
- `LKS_TT_PERF_01_v1_captions.srt`
- `LKS_IG_BRAND_01_v2_cut15.mp4`

Suggested folder structure:

- `creative/raw/` = raw recordings and source captures
- `creative/edit/` = project files and work-in-progress exports
- `creative/final/` = approved deliverables
- `creative/thumbs/` = thumbnails and poster frames
- `creative/captions/` = subtitle files

Operational rule:

- Never rename the creative ID once assigned; only increment the version or change the variant suffix. This keeps reporting, briefs, and exported files aligned.

### Creative Status Tracking Schema

Use one status per creative ID at a time:

- `Draft` = concept exists, copy/script is still being shaped
- `In Review` = asset is ready for internal feedback or stakeholder review
- `Approved` = creative is signed off and ready for trafficking
- `Live` = creative is actively running in-market
- `Paused` = creative was live but is intentionally stopped
- `Archived` = creative is retired and should not be reused without refresh

Recommended supporting fields for ops sheets:

- `Creative ID`
- `Status`
- `Owner`
- `Current Version`
- `Last Updated`
- `Launch Date`
- `Notes`

Suggested operational rules:

- Only move to `Approved` once copy, visual, CTA, and destination URL are final.
- Only move to `Live` after the exact exported asset has been trafficked.
- Use `Paused` instead of deleting rows so performance history stays intact.
- Move to `Archived` when a creative is no longer eligible for reuse.

### Campaign Naming Convention

Use this structure so campaigns, ad sets, and ads all map back to the same objective and creative system.

#### Campaign Format

`LKS_[PLATFORM]_[OBJECTIVE]_[GEO]_[AUDIENCE]_[DATE]`

Examples:

- `LKS_TT_PERF_US_BROAD_2026-05`
- `LKS_IG_BRAND_US_INTERESTS_2026-05`
- `LKS_YT_PERF_US_REMARKETING_2026-05`

Recommended campaign codes:

- **Platform:** `TT`, `IG`, `YT`
- **Objective:** `PERF`, `BRAND`
- **Geo:** `US`, `CA`, `UK`, or region label
- **Audience:** `BROAD`, `INTERESTS`, `LAL`, `RETARGETING`, `CUSTOM`
- **Date:** `YYYY-MM` for monthly grouping

#### Ad Set Format

`LKS_[PLATFORM]_[OBJECTIVE]_[AUDIENCE]_[PLACEMENT]_[OPTIMIZATION]`

Examples:

- `LKS_TT_PERF_BROAD_VERTICAL_CPA`
- `LKS_IG_BRAND_INTERESTS_REELS_VTR`
- `LKS_YT_PERF_RETARGETING_SHORTS_CPI`

Recommended ad set fields:

- **Placement:** `REELS`, `SHORTS`, `VERTICAL`, `STORIES`
- **Optimization:** `CPA`, `CPI`, `CPR`, `VTR`, `REACH`

#### Ad Name Format

`[CREATIVE_ID]_[VERSION]_[VARIANT]`

Examples:

- `LKS_TT_PERF_01_v1_hookA`
- `LKS_IG_BRAND_01_v2_master`
- `LKS_YT_PERF_01_v1_cut15`

#### Naming Rules

- Keep campaign names stable for budgeting and reporting continuity.
- Reflect targeting and optimization changes at the ad set level, not in the campaign name when possible.
- Use the exact creative ID in the ad name so reporting lines up with exported assets.
- If only the hook changes, update the variant suffix rather than creating a new creative ID.

#### Full Hierarchy Examples

- **Campaign:** `LKS_TT_PERF_US_BROAD_2026-05`
- **Ad Set:** `LKS_TT_PERF_BROAD_VERTICAL_CPA`
- **Ad:** `LKS_TT_PERF_01_v1_hookA`

- **Campaign:** `LKS_IG_BRAND_US_INTERESTS_2026-05`
- **Ad Set:** `LKS_IG_BRAND_INTERESTS_REELS_VTR`
- **Ad:** `LKS_IG_BRAND_01_v2_master`

### UTM Naming Convention

Use UTM parameters that mirror the ad platform naming structure so traffic in analytics maps directly to campaign, ad set, and creative data.

#### Recommended UTM Structure

- `utm_source` = platform source
- `utm_medium` = paid social channel type
- `utm_campaign` = campaign name
- `utm_content` = ad name or creative ID
- `utm_term` = audience or ad set detail

#### Recommended Values

- `utm_source`: `tiktok`, `instagram`, `youtube`
- `utm_medium`: `paid_social`, `paid_video`
- `utm_campaign`: exact campaign string, e.g. `LKS_TT_PERF_US_BROAD_2026-05`
- `utm_content`: exact ad string or creative ID, e.g. `LKS_TT_PERF_01_v1_hookA`
- `utm_term`: ad set or audience string, e.g. `LKS_TT_PERF_BROAD_VERTICAL_CPA`

#### UTM Rules

- Keep all UTM values lowercase in live URLs if your analytics stack is case-sensitive.
- Preserve the exact underlying naming logic even if you lowercase values for URLs.
- Do not invent separate campaign labels for UTMs; reuse the actual trafficking names.
- Use one canonical landing page URL per campaign when possible, then vary UTM values by ad set and creative.

#### Example URL Pattern

`https://lockstep.app/?utm_source=tiktok&utm_medium=paid_social&utm_campaign=lks_tt_perf_us_broad_2026-05&utm_content=lks_tt_perf_01_v1_hooka&utm_term=lks_tt_perf_broad_vertical_cpa`
